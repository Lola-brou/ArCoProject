Methodology

Our enrichment process followed a systematic four-stage approach. First, we conducted a comprehensive analysis of ArCo's knowledge graph using SPARQL queries to identify over 50 mask-related records, selecting the Okame mask, which is currently classed in arco:CulturalProperty, for enrichment due to its unique classification (dc:subject "OKAME"), sparse metadata (lacking typology, description, and cultural context), and significant cultural value in Japanese folk traditions. Through DESCRIBE queries, we confirmed specific semantic gaps including missing properties like arco:hasTypology, arco:hasCulturalContext (which we propose to create as it is not currently part of ArCo’s ontology), dc:description, and rdfs:label. For each gap, we employed tailored LLM prompting strategies: Few-shot examples generated typology and cultural context triples (testing GPT-4, Gemini, and DeepSeek outputs), chain-of-thought prompting produced a detailed 3-sentence description of Edo-period origins, and zero-shot prompting created concise multilingual labels. We validated all outputs through syntactic checks using EasyRDF and semantic verification against authoritative sources including Japan Heritage documentation and Museo delle Civiltà collection records, ensuring both technical correctness and cultural accuracy. The final selected outputs balanced ArCo's ontological conventions with the need for richer contextualization.

1. TYPOLOGY

Although several masks are present in ArCo as cultural properties (e.g., ‘MASCHERA OKAME’), none are formally linked to a relevant typology such as ‘mask’, ‘theatrical mask’, or ‘folk theatre mask’. This absence highlights a semantic gap in the ontology, which we addressed through LLM-guided RDF enrichment. The earlier DESCRIBE query confirmed that the mask lacked a formal typology classification via arco:hasTypology.

We used few-shot LLM prompting to generate RDF triples using ChatGPT, Gemini, and DeepSeek for missing information, generating RDF triples that would assign a suitable typology to the object.

Prompt: Triple Generation


You are an expert in RDF modeling for cultural heritage. Using ArCo ontology, enrich the following cultural item by assigning a more specific typology using the predicate arco:hasTypology. Provide the triple in Turtle syntax.

Target cultural item:
<https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901>

Context:
This is the "Maschera Okame," a Japanese mask used in folk theater, often representing a cheerful, plump-faced woman symbolizing good fortune and humor.

Generate one RDF triple that links this object to a specific typology.

Examples:

            1. <https://w3id.org/arco/resource/MovableCulturalProperty/1500556890> arco:hasTypology <https://w3id.org/arco/resource/Typology/brass-wind-instrument> .
            2. <https://w3id.org/arco/resource/MovableCulturalProperty/1200765421> arco:hasTypology <https://w3id.org/arco/resource/Typology/percussion-instrument> .

Now generate the triple for the mask.


Output: Triple Generation

Proposed typology RDF triples for the Okame mask from different LLMs.

The models returned the following proposed triples:

  • ChatGPT: arco:hasTypology <.../folk-theatre-mask>
  • DeepSeek: arco:hasTypology <.../theatrical-mask>
  • Gemini: arco:hasTypology <.../Japanese-mask>

We then verified whether these proposed typologies already existed in the ArCo knowledge graph. None of them are currently defined as instances of arco:Typology, indicating that the LLMs had generated hypothetical URIs based on logical linguistic structure — but not on existing ontology terms.

This result demonstrates both the power and limitation of LLM-based enrichment: While the models can generate semantically reasonable suggestions, human verification is essential to ensure ontology compliance.

Ultimately, we selected the URI folk-theatre-mask as the most semantically accurate for the Okame mask. DeepSeek’s theatrical-mask was potentially too broad, while Gemini’s Japanese-mask was categorically incorrect for a typology, fitting better within a cultural context. We then used ChatGPT’s URI in a CONSTRUCT query to simulate the enrichment as it was the most semantically accurate:

        <https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901> 
            arco:hasTypology <https://w3id.org/arco/resource/Typology/folk-theatre-mask> .

We validated the triples using EasyRDF’s Turtle parser, confirming their syntactic correctness and readiness for integration into the knowledge graph.

Cultural context RDF triples generated via LLM.

The proposed new typology URI https://w3id.org/arco/resource/Typology/folk-theatre-mask addresses a semantic gap in ArCo's ontology by providing a precise classification for folk theater masks like the Okame mask, which currently lack a distinct category. While ArCo includes broad terms like "maschera teatrale"@it, it fails to distinguish folk traditions (e.g., Japanese Kagura, Italian Commedia dell'arte) from classical or ritual mask types. This enrichment enables accurate scholarly queries while maintaining compatibility with ArCo's structure.

2. Cultural Context

For the Okame mask, we enriched its metadata by introducing the property arco:hasCulturalContext as a custom extension to capture the cultural and geographic background of heritage objects. Although the original ArCo record did not specify the mask’s country of origin, purpose, or provenance, adding cultural context helps situate the object within its cultural narrative and significance.

Because cultural context is a broad and open-ended concept, we decided to use few-shot prompting to guide the LLMs to produce relevant, precise cultural context descriptions without veering into unrelated details. This approach provided a more robust and reliable output than zero-shot prompting, while not requiring the deeper reasoning processes of chain-of-thought prompting.

While the ArCo ontology currently does not define a dedicated property explicitly for cultural context, we proposed using a custom predicate named arco:hasCulturalContext to capture this important aspect. This property is intended to express the cultural and geographic background of heritage objects in a human-readable way.

Since the subject of cultural context is more open-ended than typology, the constraints given in the few-shot prompts are so that the LLMs would not extract irrelevant information.



Prompt: Cultural Context

You are an expert in RDF modeling for cultural heritage and linked open data. Your task is to generate RDF triples in Turtle syntax describing the cultural context of heritage objects using the predicate arco:hasCulturalContext.

Here are some examples related to the Maschera Okame mask:

Example 1:
Resource: https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901
Triple:
<https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901> arco:hasCulturalContext "The Maschera Okame is a traditional Japanese folk theatre mask originating from Japan, symbolizing good fortune and happiness in local performing arts."@en .

Example 2:
Resource: https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901
Triple:
<https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901> arco:hasCulturalContext "A papier-mâché mask from Japanese Noh theatre representing a joyful female character."@en .

Now, generate an RDF triple in Turtle syntax for the following resource, focusing only on its cultural and geographic context:

Resource: https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901


Output: Cultural Context

DeepSeek generated the best triple in this case. Gemini mentioned “Japan” and “Japanese” twice, being redundant, and was missing the declaration of what ArCo is. ChatGPT’s results were adequate but mixed the English and Italian “maschera” which could be confusing.

Validation
We validated the triples using EasyRDF’s Turtle parser, confirming their syntactic correctness and readiness for integration into the knowledge graph.

Cultural Context Validation

We then queried our triple linking the Okame mask to the cultural context, demonstrating how our enrichment could be programmatically added to the knowledge graph, similarly to the typology portion of our project. Unlike typology whose query uses a URI from ArCo's controlled vocabulary, https://w3id.org/arco/resource/Typology/, the hasCulturalContext property uses a literal string: <.../1208539901> arco:hasCulturalContext "A traditional Japanese folk mask..."@en to create human-readable descriptions rather than linked data URIs.

This approach prioritizes human-readable descriptions over formal linked data URIs. If this were a real-world knowledge graph, minting dereferenceable URIs for cultural contexts would add value by enabling interoperability, provenance tracking, and integration with other datasets. However, for the purposes of this project, literals are pragmatically sufficient—and their use also allows us to critique ArCo’s gaps in modeling contextual knowledge, which often relies on textual descriptions rather than structured vocabularies.

3. Description
Before we began the prompt, we asked ChatGPT to provide us with information about the Okame mask, to make sure that it aligned with the information we found on Japanese cultural websites, and also to give ourselves a better understanding of the material we are enriching.

In researching the other masks in the ArCo ontology, we noticed that some have long, detailed descriptions, such as the Carnevalesca di Cento Mask and the Egetmann-Umzug Mask. However, the majority of masks in the ArCo ontology have no descriptions. Through research done on the collections of the Museo delle Civiltà where the Okame mask from the ontology is held, as well as other online resources, we aimed to add relevant https://dati.beniculturali.it/lodview-arco/resource/DemoEthnoAnthropologicalHeritage/1200531913.htmldepth that was not redundant to the information we propose for arco:hasCulturalContext.

We felt that using the Chain of Thought (CoT) prompt would be the best for a longer, more detailed description, meant for human comprehension, than a few-shot or zero-shot prompt could provide. This was based on information we gathered when researching the descriptions of other masks in ArCo.

We also asked the three LLMs to describe the Okame Mask, in order to prompt an LLM to generate valuable knowledge:


Prompt: Okame Mask Description

                Generate a 3-5 sentence dc:description for the Okame Mask in Turtle syntax. Include:
                1. Origin & History: Japanese folk tradition (Edo period?).
                2. Material/Technique: Painted papier-mâché (from ArCo’s data).
                3. Visual Traits: Round face, red cheeks, smiling expression.
                4. Symbolism: Represents Okame (Otafuku), goddess of joy/fertility.
                5. Usage: Worn in festivals (e.g., Kagura) and folk theater.
                Avoid redundancy with arco:hasCulturalContext. Output format:
                turtle
                <https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901>
                    dc:description \"[text]\"@en .
                ```

Output: Okame Mask Description



Reasoning

The optimal description was selected through rigorous evaluation against three criteria: (1) depth (encompassing history, symbolism, and usage), (2) accuracy (cross-referenced with authoritative sources like Japan Heritage), and (3) readability (fluency without redundancy).

ChatGPT’s output emerged as the strongest candidate for its comprehensive yet concise synthesis of the Okame mask’s Edo-period origins, papier-mâché construction, and role in Kagura festivals, while distinguishing it from formal Noh theater traditions.

This extended description aligns with ArCo’s stylistic conventions, mirroring the granularity seen in records for comparable artifacts. It delivers critical value to researchers by clarifying the mask’s historical context and performative function—filling gaps left by broader properties like hasCulturalContext, balancing detail with brevity, avoiding redundancy while enriching the knowledge graph’s semantic utility.

4. Label Modification

The current label, "MASCHERA"@it, is overly generic and fails to capture the unique identity of the Okame mask within ArCo’s knowledge graph. As a result, this mask—a culturally significant Japanese artifact with distinct iconography and purpose—is conflated with other masks in queries and semantic searches. Additionally, the vast majority of masks are catalogued under "MASCHERA DELL'OPERA CINESE", "MASCHERA TEATRALE"@it, or simply "MASCHERA"@it (such as the Okame mask), with none making the distinction between each other, despite all being completely different artifacts. Our proposal to enrich the label to "Maschera Okame"@it serves two critical purposes:

  • Enhanced Searchability: A more specific label allows for precise retrieval in SPARQL queries (e.g., filtering masks by rdfs:label "Maschera Okame"@it). This is particularly valuable for researchers studying Japanese folk art or performance, as it distinguishes the Okame mask from broader categories like theatrical or ritual masks.
  • Cultural Precision: The Okame mask represents a specific archetype in Japanese folklore (the goddess Okame/Otafuku, symbolizing joy and fertility). The revised label aligns with scholarly terminology and avoids the ambiguity of the original label, which could refer to any mask in the collection.

For this task, zero-shot prompting is both efficient and appropriate for its simplicity since label updates require minimal contextual reasoning compared to generating descriptions or cultural contexts. This should also be a straightforward task for the inherent consistency in the task. The prompt only needs to enforce language (@it) and specificity ("Okame"), which LLMs handle reliably without examples, making it difficult to garner confusion. Unlike open-ended tasks, label generation has fewer opportunities for hallucination—the output is a direct linguistic modification.

The approach mirrors ArCo’s existing practices, where labels are often concise noun phrases (e.g., "Maschera teatrale"@it). By refining the label to reflect the artifact’s cultural identity, we bridge a gap in the ontology’s terminological precision without introducing complexity.

Prompt: Label Enrichment


        Generate an improved RDF label for an Italian cultural heritage record.
        Current label: 'MASCHERA'@it (too generic).
        Context: This is a Japanese Okame mask (folk theater, symbolizes joy).
        Requirements:
        Language: Italian (@it). And English (@en).
        Specificity: Include 'Okame'.
        Format: Turtle syntax.
        Output only the triple, no explanations.


Output: Label Enrichment


Explanation

We propose the use of DeepSeek’s enriched label proposal 

        <https://w3id.org/arco/resource/DemoEthnoAnthropologicalHeritage/1200539901>
            rdfs:label "Maschera Okame"@it, "Okame Mask"@en ;
            dc:title "Maschera Okame"@it, "Okame Mask"@en .

as we felt it added the enhanced specificity required to contextualize the Okame mask, while still meeting ArCo’s labeling conventions (use of 1-3 word noun phrases primarily in Italian). The introduction of multilingual support (Italian and English) improves accessibility for international researchers such as ourselves and aligns with the mask’s existing dc:subject "OKAME" classification. The minimalist approach—using concise "Maschera Okame"@it and "Okame Mask"@en—intentionally avoids redundancy with the richer contextual data in arco:hasCulturalContext and the newly added dc:description. For consistency, this labeling strategy should be cross-referenced with the mask’s other enriched properties (typology, cultural context) to present a unified semantic profile.

Validation

Once again we validated our triple using EasyRDF to ensure syntactic correctness before documenting them for ArCo. When checking the multilingual label and title updates, the parser correctly recognized four distinct triples from our Turtle-formatted input. Each language variant (@it, @en) for both rdfs:label and dc:title is treated as a separate triple in RDF. This mirrors ArCo’s real-world requirements, where multilingual labels improve accessibility for Italian and English users, and titles and labels often duplicate values (as seen in other ArCo records).

Earlier, properties like arco:hasTypology or dc:description parsed as single triples because they use one unambiguous value (e.g., a URI for typology, a monolingual literal for description) and align with ArCo’s convention of storing classifications and contextual data without language variants. Our labels follow ArCo’s multilingual pattern (e.g., "MASCHERA TEATRALE"@it). Finally, the parser’s output confirms we’ve avoided syntax errors in our proposal. The four triples here versus one elsewhere reflects intentional design choices for different predicate types. This validation step ensures our enrichments are technically sound before proposing them to ArCo maintainers.
